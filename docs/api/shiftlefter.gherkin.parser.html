shiftlefter.gherkin.parser

Recursive-descent parser for Gherkin token sequences.

Consumes tokens produced by the lexer and produces AST nodes (Feature, Scenario, Background, Rule, Step, etc.). Collects errors without throwing — returns {:ast [...] :errors [...]}.

Internal to the parser pipeline; use gherkin.api/parse-string for the public API.

*assert-advancing*

dynamic

Development-time guard for detecting non-advancing parser loops. When true, throws if a ‘many’ parser loop fails to advance the token stream. Set to true during testing to catch infinite loop bugs early.

*dialect*

dynamic

Current dialect for parsing. Bind to a dialect map (from dialect/get-dialect) to parse non-English Gherkin. Defaults to nil (uses English-only fallback).

parse

(parse token-seq)

Parse a sequence of tokens into a Gherkin AST.

Returns a map with: - :tokens — the realized token vector - :ast — vector of AST nodes (Feature, Blank, Comment records) - :errors — vector of parse errors (empty if successful)

Example: (parse (lex “Feature: Test\n Scenario: Demo”))
