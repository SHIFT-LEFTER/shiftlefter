shiftlefter.adapters.registry

Adapter registry for capability management.

Maps adapter names (like :etaoin) to factory/cleanup functions, enabling pluggable capability creation.

Registry Structure

An adapter entry has: - :factory — function that takes config, returns capability - :cleanup — function that takes capability, cleans up resources

Usage

;; Get adapter by name
(get-adapter :etaoin)
;; => {:factory #'shiftlefter.adapters.etaoin/create-browser
;;     :cleanup #'shiftlefter.adapters.etaoin/close-browser}

;; Create capability
(create-capability :etaoin {:headless true})
;; => {:ok {:browser ... :etaoin-driver ...}}

;; Cleanup capability
(cleanup-capability :etaoin capability)
;; => {:ok :closed}

cleanup-capability

(cleanup-capability adapter-name capability)(cleanup-capability adapter-name capability registry)

Clean up a capability using the named adapter.

Parameters: - adapter-name: Keyword identifying the adapter - capability: Capability to clean up (as returned by create-capability) - registry: Optional registry map (defaults to default-registry)

Returns: - Success: Result from adapter’s cleanup (typically {:ok :closed}) - Error: {:error {:type :adapter/unknown …}} if adapter not found - Error: Result from cleanup on failure

Examples:

(cleanup-capability :etaoin {:browser b :etaoin-driver d})
;; => {:ok :closed}

create-capability

(create-capability adapter-name config)(create-capability adapter-name config registry)

Create a capability using the named adapter.

Parameters: - adapter-name: Keyword identifying the adapter - config: Configuration map passed to the adapter’s factory - registry: Optional registry map (defaults to default-registry)

Returns: - Success: Result from adapter’s factory (typically {:ok {…}}) - Error: {:error {:type :adapter/unknown …}} if adapter not found - Error: Result from factory on creation failure

Examples:

(create-capability :etaoin {:headless true})
;; => {:ok {:browser #EtaoinBrowser{...} :etaoin-driver {...}}}

get-adapter

(get-adapter adapter-name)(get-adapter adapter-name registry)

Look up an adapter by name.

Parameters: - adapter-name: Keyword identifying the adapter (e.g., :etaoin) - registry: Optional registry map (defaults to default-registry)

Returns: - Success: {:factory fn :cleanup fn} - Error: {:error {:type :adapter/unknown :adapter name :known …}}

Examples:

(get-adapter :etaoin)
;; => {:factory #function :cleanup #function}

(get-adapter :unknown)
;; => {:error {:type :adapter/unknown :adapter :unknown :known [:etaoin]}}
