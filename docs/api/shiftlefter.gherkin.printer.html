shiftlefter.gherkin.printer

Lossless printer that reconstructs original input from tokens.

This is a ‘do nothing’ printer - it proves the lexer preserves all information needed for byte-for-byte roundtrip without any formatting or transformation.

Policy B: Parse errors => refuse to format/rewrite, return error map.

canonical

(canonical input)

Format a string to canonical style.

Returns formatted string on success. Throws on parse errors.

fmt-canonical

(fmt-canonical path)

Format a file to canonical style.

Returns: - {:status :ok :path path :output formatted-string} on success - {:status :error :reason :parse-errors :details …} on parse failure - {:status :error :reason :io/utf8-decode-failed …} if file is not valid UTF-8 - {:status :error :reason :io/file-not-found …} if file doesn’t exist

fmt-check

(fmt-check path)

Check if file round-trips perfectly via lex -> print-tokens.

Returns: - {:status :ok :path path} if roundtrip matches exactly - {:status :error :reason :parse-errors :details …} if parse fails - {:status :error :reason :mismatch :path path …} if bytes differ - {:status :error :reason :io/utf8-decode-failed …} if file is not valid UTF-8 - {:status :error :reason :io/file-not-found …} if file doesn’t exist

format-canonical

(format-canonical ast)

Format AST to canonical Gherkin style.

Canonical style: - 2 spaces indentation - Aligned table columns - No trailing whitespace - Single blank line between sections

Returns: - {:status :ok :output formatted-string} on success

print-tokens

(print-tokens tokens)

Concatenate :raw fields from tokens to reconstruct original input. Excludes the EOF token (which has empty :raw).
