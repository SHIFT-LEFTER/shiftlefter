shiftlefter.subjects.profile

Subject profile directory and metadata management.

Subjects are named browser profiles stored at ~/.shiftlefter/subjects/<name>/. Each subject has: - A Chrome user-data-dir at chrome-profile/ - A metadata file browser-meta.edn tracking connection state

Directory Structure

~/.shiftlefter/subjects/
└── finance/
    ├── browser-meta.edn
    └── chrome-profile/
        └── (Chrome user data)

browser-meta.edn Shape

{:debug-port 9222
 :user-data-dir "/Users/x/.shiftlefter/subjects/finance/chrome-profile"
 :chrome-pid 12345
 :stealth true
 :created-at "2026-01-08T12:00:00Z"
 :last-connected-at "2026-01-08T12:00:00Z"}

delete-profile!

(delete-profile! subject-name)

Delete an entire subject profile directory.

Removes the subject directory and all contents (Chrome profile, metadata). Returns true if deleted, false if directory didn’t exist.

Examples:

(delete-profile! :finance)
;; => true

ensure-dirs!

(ensure-dirs! subject-name)

Create subject profile directories if they don’t exist.

Creates: - ~/.shiftlefter/subjects/<name>/ - ~/.shiftlefter/subjects/<name>/chrome-profile/

Returns the profile directory path.

Examples:

(ensure-dirs! :finance)
;; => "/Users/x/.shiftlefter/subjects/finance"

load-browser-meta

(load-browser-meta subject-name)

Load browser metadata for a subject.

Returns the metadata map, or nil if file doesn’t exist.

Examples:

(load-browser-meta :finance)

save-browser-meta!

(save-browser-meta! subject-name meta)

Save browser metadata for a subject.

Uses atomic write (temp file + rename) to prevent corruption. Ensures parent directories exist.

Automatically adds/updates :last-connected-at timestamp. Preserves existing :created-at if file already exists.

Examples:

(save-browser-meta! :finance {:debug-port 9222
                               :chrome-pid 12345
                               :stealth true})
